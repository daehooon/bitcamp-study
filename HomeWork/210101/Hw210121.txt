- 학습 목표 달성 확인 목록
    - [] UML이 무엇이고 어떤 용도로 사용하는지 아는가?
        : 프로그램 설계를 표현하기 위해 사용하는,
          주로 그림으로 된 표기법
    - [] 의존 객체(dependency)가 무엇인지 이해하는가?
        : 사용당하는 객체
    - [] GRASP 패턴이 무엇인지 아는가?
        : 객체의 책임 부여 원칙, 클래스를 정의하는 바람직한 방법
    - [] Information Expert 패턴에 대해 이해하는가?
        : 정보를 사용하는 쪽에 책임을 맡겨라
    - [] domain, vo(value object), dto(data transfer object) 클래스의 의미를 이해하는가?
        : 개발자가 시스템 목적에 맞춰서 새로 정의한 데이터 타입
    - [] 스태틱 필드의 사용 범위와 그 한계를 이해하는가?
        : 사용범위는 코드에 public을 작성하면 패키지간에도 사용 가능하고
          같은 기능을하는 이름만 다른 클래스를 복제하여 사용 가능
          한계는 복제본들의 기능을 추가, 변경, 삭제하려면 복제한 클래스들의 코드를 하나하나 전부 고쳐야한다.
    - [] '기술 부채'가 무엇인지 설명할 수 있는가?
        : 일반적인 부채가 이자를 내고 돈을 쓰는 시점을 당기는 것처럼
          기술 부채는 기술적으로 해결되어야 할 문제들을 뒤로 미루고,
          비즈니스 문제를 해결하는 시점을 당기는 것이다.
